Verify data integrity.

  • Nahashon Muiru
  • Jan 1, 2022

Files and Software conveyed over the internet or moved from one storage location to another might be susceptible to alterations that can manipulate the information stored in them or trigger security vulnerbilities. These alterations can be attributed to an individual or to just computer errors.

The checksum (message digest), is a digit or a character derived by applying a suitable algorithm to some data, used to check whether errors have occured in transmission, storage or data entry.

Cryptographic Hashing Algorithms

The checksum value acts as a fingerprint for a file or a set of files. Any slight difference in the values at the source and destination is indicative of tampering. However if they match the files are identical. There are multiple checksum algorithms present that can be used, some these include;

  • SHA-256
  • SHA-384
  • SHA-512
  • MD5

    • How to

    To verify the credibility of our file, we simply run the command, say, md5sum, with the file(s) to be validated as the parameter. md5sum file(s)



    GUI tools such as GtkHash are also available for the same purpose.

    Some of the reasons the value can be different at source and destination is due to;
  • Disk space issues
  • Data corruption
  • Network interruption
  • Third party interference
